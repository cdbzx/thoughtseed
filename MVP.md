# 灵芽（ThoughtSeed）MVP 说明

## 1. MVP 目标
用最小成本验证三件事：
1. 用户是否愿意高频记录灵感
2. AI 是否能提升碎片灵感的可用性
3. 复习机制是否能让旧灵感重新被使用

---

## 2. MVP 核心原则
- 功能少，但闭环完整
- 优先验证价值，不追求功能面面俱到
- 优先做出“记录 → 补全 → 复习”的最短路径

---

## 3. MVP 必做功能

### 3.1 快速记录
- 文本记录
- 语音转文字
- 自动时间戳
- 一键保存

### 3.2 AI 补全
- 一键展开
- 自动摘要
- 自动标签

### 3.3 灵感列表与检索
- 时间流展示
- 搜索
- 标签筛选
- 收藏

### 3.4 复习机制
- 每日回顾 3~5 条旧灵感
- 高潜力灵感推荐
- 继续发展 / 忽略 / 稍后处理

---

## 4. MVP 暂不做
- 团队协作
- 社交功能
- 知识图谱
- 多端原生 App
- 插件系统
- 高级权限体系
- 复杂工作流

---

## 5. MVP 用户闭环
1. 用户记录一条灵感
2. 系统保存并打时间标签
3. 用户点击 AI 展开
4. 系统输出结构化内容
5. 次日或后续系统推送复习
6. 用户决定继续发展或忽略

---

## 6. MVP 页面范围
- 首页
- 快速记录页
- 灵感列表页
- 灵感详情页
- 复习页
- 设置页

---

## 7. MVP 成功标准
- 用户能在 3 秒内完成记录
- AI 展开结果对用户有实际帮助
- 用户愿意回看系统推荐的旧灵感
- 至少有一部分灵感进入转化动作

---

## 8. 技术建议
- 前端：Next.js + Tailwind
- 后端：Node.js API
- 数据库：SQLite
- AI：OpenAI compatible + Ollama
- 语音：先接第三方 STT
