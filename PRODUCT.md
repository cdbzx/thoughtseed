# 灵芽（ThoughtSeed）产品文档

## 1. 项目概述

### 1.1 项目名称
- 中文名：灵芽
- 英文名：ThoughtSeed

### 1.2 一句话定位
一个开源的 AI 灵感管理工具，帮助用户快速捕捉一闪而过的念头，并通过 AI 自动补全、关联与复习，让碎片灵感逐步成长为可执行的想法、内容和项目。

### 1.3 核心价值主张
让灵感从一闪而过，变成可复用、可回顾、可转化的资产。

### 1.4 Slogan
- 中文：让每个念头都有生长的机会
- 英文：Every idea starts as a seed

---

## 2. 背景与问题定义

### 2.1 背景
很多人并不缺灵感，缺的是：
- 在灵感出现的瞬间，能够低成本记录下来
- 在灵感不完整时，能被系统辅助补全
- 在灵感沉底后，能被重新唤醒和继续发展

### 2.2 当前常见问题
1. 灵感出现很快，消失也很快
2. 记录工具打开慢、操作重、分类复杂
3. 记录下来的内容往往只是半句话或关键词
4. 记录后缺少整理与复习机制，最终沉底
5. 灵感和行动脱节，无法转化为创作或项目

### 2.3 产品要解决的核心问题
- 如何让用户在 3 秒内完成灵感捕获
- 如何把碎片灵感补全成结构化内容
- 如何让旧灵感被重新发现、被继续发展

---

## 3. 产品定位

### 3.1 产品定义
灵芽（ThoughtSeed）是一个围绕“灵感生命周期”设计的开源产品，覆盖：
- 快速捕获
- AI 补全
- 智能关联
- 复习唤醒
- 转化输出

### 3.2 产品边界
本产品不是传统通用笔记工具，也不是泛化聊天机器人，而是专门面向“碎片灵感”与“短瞬思考”的管理系统。

### 3.3 目标用户
优先用户群：
- 产品经理
- 独立开发者
- 创作者 / 写作者 / 内容生产者
- 创业者
- 学生 / 研究者
- 经常有碎片化想法、但不善于整理的人

---

## 4. 差异化亮点

### 4.1 极低摩擦记录
- 支持文字、语音、图片、链接等快速输入
- 不要求先分类、先命名、先整理
- 以“先收集，再加工”为原则

### 4.2 AI 自动补全半成品灵感
- 把半句话、关键词、模糊念头补全成结构化内容
- 自动提炼问题、目标用户、价值、方向和下一步动作

### 4.3 智能复习机制
- 定期回顾旧灵感
- 间隔复习高潜力内容
- 唤醒被遗忘但有价值的记录

### 4.4 灵感关联与组合
- 自动发现相似灵感
- 合并重复想法
- 提示可组合的点子
- 形成“灵感网络”而非孤立条目

### 4.5 从灵感到行动
- 将灵感转化为任务、文章草稿、项目卡片、创意方向
- 让记录不止停留在收藏层面

---

## 5. 产品目标

### 5.1 短期目标（MVP）
- 验证用户是否愿意高频记录灵感
- 验证 AI 补全是否能提高灵感可用性
- 验证复习机制是否能提升旧灵感重访率

### 5.2 中期目标
- 建立灵感之间的自动关联能力
- 提升从灵感到内容/任务的转化率
- 完成本地优先与多模型支持的开源产品能力

### 5.3 长期目标
成为创作者、产品经理、独立开发者的“灵感操作系统”。

---

## 6. 核心模块设计

### 6.1 Capture：快速捕获
目标：让用户用最低成本记录灵感。

支持输入：
- 文字输入
- 语音输入
- 图片/截图
- 网页链接分享
- 快捷键唤起
- 移动端快捷入口

记录字段：
- 原始内容
- 时间戳
- 来源（文本 / 语音 / 图片 / 链接）
- 标签（手动或 AI 自动）

设计原则：
- 先记录，再整理
- 不打断用户思路

### 6.2 Expand：AI 补全与整理
目标：把碎片灵感变成可继续思考的素材。

支持动作：
- 展开成完整想法
- 提炼核心观点
- 生成标题
- 自动打标签
- 分类归档
- 合并相似灵感
- 生成下一步建议

输出结构建议：
- 核心问题
- 目标用户
- 使用场景
- 产品价值
- 可行方向
- 下一步动作

### 6.3 Review：智能复习与唤醒
目标：让灵感重新被看见，而不是沉底。

复习方式：
- 每日随机回顾
- 高潜力灵感回顾
- 被遗忘灵感回顾
- 同主题灵感回顾
- 间隔重复复习

复习动作：
- 继续发展
- 忽略/稍后再看
- 合并到主题
- 转为任务
- 转为文章草稿

## 7. MVP 功能清单

### 7.1 核心功能
1. 快速记录
   - 文本输入
   - 语音转文字
   - 自动记录时间
   - 一键保存

2. AI 处理
   - 一键展开灵感
   - 一键总结要点
   - 自动生成标签

3. 灵感管理
   - 时间流列表
   - 搜索
   - 标签筛选
   - 收藏/置顶

4. 复习系统
   - 每日回顾
   - 高潜力灵感推荐
   - 相似灵感提示
   - 继续发展操作

### 7.2 第一版暂不纳入
- 团队协作
- 社交社区
- 复杂知识图谱
- 多端原生同时开发
- 复杂插件系统
- 大量模型深度适配

## 9. 核心用户流程

### 9.1 记录流程
用户突然有想法 → 打开灵芽 → 输入一句话或说一句话 → 保存 → 系统自动打时间与标签

### 9.2 AI 补全流程
用户选择某条灵感 → 点击“展开” → AI 生成结构化内容 → 用户编辑/确认

### 9.3 复习流程
系统每天推荐若干旧灵感 → 用户查看 → 选择继续发展/忽略/合并/转任务

---

## 10. 用户故事

1. 作为一个经常突然冒出想法的人，我希望能在几秒内记下灵感，这样我不会因为记录成本太高而错过好点子。
2. 作为一个记录了很多碎片想法的人，我希望 AI 帮我补全整理，这样我能更容易继续思考和行动。
3. 作为一个容易遗忘旧想法的人，我希望系统能定期唤醒过去的灵感，这样记录不会长期沉底。
4. 作为一个创作者或独立开发者，我希望能把灵感快速转成任务、文章或项目方向，这样灵感能真正进入执行。

---

## 11. 关键指标（KPI）

### 11.1 核心指标
- 日均记录条数
- 记录成功率
- AI 补全使用率
- 7 日内旧灵感重访率
- 灵感转化率（转任务 / 转文章 / 转项目）
- 次日留存 / 7日留存

### 11.2 最关键的两个指标
1. 用户是否持续记录
2. 旧灵感是否被重新激活

---

## 12. 风险与应对

### 12.1 风险：沦为普通笔记工具
应对：突出快速记录、AI 补全、复习唤醒三大核心价值。

### 12.2 风险：AI 补全结果空泛
应对：采用固定结构补全，而不是泛化聊天式回答。

### 12.3 风险：复习打扰感强
应对：复习频率可调，支持“仅提醒高潜力灵感”。

### 12.4 风险：记录容易，转化困难
应对：增加“转为任务 / 草稿 / 项目”的出口。

## 13. 技术路线建议

### 13.1 前端
- Web 优先
- 移动端先做 PWA 或简单 H5
- 后续再补原生 App

建议技术：
- React / Next.js
- Tailwind CSS

### 13.2 后端
- API 服务
- 用户数据、灵感数据、标签、复习记录管理

建议技术：
- Node.js / NestJS / Next.js API

### 13.3 AI 层
- 支持 OpenAI 兼容接口
- 支持本地 Ollama
- 支持模型可切换
- 后续支持 embedding 相似度检索

### 13.4 存储
- SQLite 起步即可
- 后续切 PostgreSQL

### 13.5 语音能力
- 第一版接第三方语音转文字服务
- 后续支持本地 Whisper

---

## 14. 第一版开源卖点
如果发布到 GitHub，建议突出以下卖点：
- 本地优先 / 可自托管
- 支持多模型接入
- 快速记录灵感
- AI 自动整理碎片想法
- 间隔复习与唤醒机制
- 数据可导出
- 对创作者 / 独立开发者友好

---

## 15. 结论

这个项目方向成立，且比“再做一个聊天机器人”更有辨识度。

真正的核心不是“记灵感”，而是：

> 让灵感从一闪而过，变成可复用、可回顾、可转化的资产。

灵芽（ThoughtSeed）如果聚焦在：
- 快速捕获
- AI 补全
- 智能关联
- 复习唤醒
- 转化输出

就有机会做成一个差异化明显的开源产品。
